## 프리미티브 타입 종류와 값의 범위 그리고 기본 값

- 모든 자료형은 값을 지정하지 않으면 기본값을 사용한다.
- 정수형
    - byte
        - 1바이트, $(-2^7 \sim 2^7-1)$
        - 기본값: 0
    - short
        - 2바이트, $(-2^{15} \sim 2^{15} - 1)$
        - 기본값: 0
    - int
        - 4바이트, $(-2^{31} \sim 2^{31}-1)$
        - 기본값: 0
    - long
        - 8바이트, $(-2^{63} \sim 2^{63}-1)$
        - 기본값: 0
    - char
        - 2바이트, $(0 \sim 2^{16} - 1)$
        - 기본값: \u0000
- 소수형
    - 간단한 계산에서는 사용해도 무방하나 돈 계산과 같이 중요한 부분에서는 이 타입을 사용해서는 안된다.
    - 32비트와 64비트로 제공할 수 있는 범위를 넘어서면 그 값의 정확성을 보장하지 않기 때무
    - float
        - 4바이트
        - 기본값: 0.0
    - double
        - 8바이트
        - 기본값: 0.0
- boolean
    - 기본값: false

## 프리미티브 타입과 레퍼런스 타입

### 프리미티브 타입(Primitive Type)

- byte, short, int, long, char, float, double, boolean

### 레퍼런스 타입(Reference Type)

- `new`를 사용해서 객체를 생성하는 모든 것
    - String은 new 없이 생성 가능

## 리터럴

### 리터럴이란?

- 프로그램에서 값을 직접 설정하여 표현하는 것
- 정수 리터럴
    
    ```java
    int a = 15; // 10진수 리터럴
    int b = 015; // 8진수 리터럴
    int c = 0x15; // 16진수 리터럴
    int d = 0b0111; // 2진수 리터럴
    ```
    
- long 타입 리터럴
    - 숫자 뒤에 `L`이나 `l`을 붙여서 표시
    
    ```java
    long a = 0L;
    ```
    
- 실수 리터럴
    - 소수점 형태나 지수 형태로 표현한 것
    - default는 double 타입으로 컴파일
    
    ```java
    float a = 0.12f;
    double b = 12E-2;
    ```
    
    - 명시적으로 뒤에 `f`나 `D`를 붙여서 표시하기도 한다.
    - float 타입으로 컴파일 하고 싶다면 무조건 표시해야 한다. (double은 생략 가능)
- 문자 리터럴
    - `' '`를 사용하여 문자 표현
        
        ```java
        char a = 'A';
        char b = "아현";
        char c = \uae01; // 유니코드 값
        ```
        
- 문자열 리터럴
    - `" "`를 사용하여 문자열 표현
        
        ```java
        String a = "abcd";
        ```
        
- 논리 타입 리터럴
    - C와 달리 0과 1을 참, 거짓으로 사용할 수 없다.
        
        ```java
        boolean a = true;
        boolean b = 1; // Compile Error
        ```
        

## 변수 선언 및 초기화하는 방법

### 초기화란?

- 변수를 사용할 수 있는 상황을 만들기 위해 변수와 값을 지정하는 것

### 변수 선언 및 초기화

```java
int a = 10; // 프리미티브 타입 초기화
Car c = new Car(); // 레퍼런스 타입 초기화
String d = "hello"; // String 타입 초기화 (1)
STring e = new String("hello"); // String 타입 초기화 (2)
```

## 변수의 스코프와 라이프타임

```java
class Variables {
	int a; // 인스턴스 변수, Non-static Fields
	static int b; // 클래스 변수, static Fields, 공유 변수
	
	void method(int d) { // 매개 변수
		int c; // 지역 변수
	}
}
```

- 자바에서의 변수의 종류는 4가지
    - 지역 변수(local variable)
        - 중괄호 내에서 선언된 변수
        - 지역 변수를 선언한 중괄호 내에서 유효
    - 매개 변수(parameters)
        - 메서드에 넘겨주는 변수
        - 메서드 호출 시 라이프 시작, 메서드가 종료되면 소멸
    - 인스턴스 변수(instance variable)
        - 메서드 밖, 클래스 안에 선언된 변수
        - `static`이라는 예약어가 붙지 않는다.
        - 객체가 생성될 때 라이프 시작, 객체를 참조하고 있는 다른 객체가 없다면 종료
    - 클래스 변수(class variable)
        - 인스턴스 변수처럼 메서드 밖, 클래스 안에 선언된 변수
        - `static`이라는 예약어가 붙는다.
        - 클래스가 처음 호출될 때 라이프 시작, 자바 프로그램이 끝날 때 종료

## 타입 변환, 캐스팅 그리고 타입 프로모션

### **타입 변환**

- 데이터 타입을 다른 데이터 타입으로 변환하는 것을 말한다.
- 타입 변환의 종류에는 **자동 타입 변환(Promotion)**과 **강제 타입 변환(Casting)**이 있다.

### **자동 타입 변환(Promotion)**

- 작은 크기를 가지는 타입이 큰 크기를 가지는 타입에 저장될 때 발생한다.
- 메모리의 크기로 큰 크기의 타입과 작은 크기 타입을 구분한다.

<aside>
💡 데이터 타입 크기 비교

byte(1) < short(2) < int(4) < long(8) < float(4) < double(8)

</aside>

```java
byte byteValue = 10;

int intValue = value; // 자동 타입 변환이 일어난다.
```

- 자동 타입 변환에서의 예외는 char의 범위는 0부터 시작하기 때문에 음수가 저장될 수 있는 byte 타입을 char 타입으로 자동 변환시킬 수 없다.

### **강제 타입 변환(Casting)**

- 강제적으로 큰 타입의 데이터를 작은 타입의 데이터로 쪼개서 저장하는 것을 말한다.
- 강제 타입 변환은 `()`를 이용하여 변환한다.

```java
int intValue = 103029770;

byte byteValue = (byte) intValue; // 강제 타입 변환
```

- intValue를 byteValue에 저장할 때 원래 값이 보존되지 않고, byte 타입의 메모리 크기(1byte)만큼 값을 가져가게 된다. 그리고 나머지는 버려진다.
- 강제 타입 변환을 하기 전에 우선 안전하게 값이 보존될 수 있는지부터 생각하면 좋을 것 같다.

## 1차 및 2차 배열 선언하기

### **1차 배열 선언 방법**

1. **타입[] 변수;**
    
    ```java
    int[] num;
    ```
    
2. 타입 변수[];
    
    ```java
    int num[];
    ```
    
- 위 두 가지 방법이 있는데 보통 1번을 많이 쓴다.
- 2번은 C언어 느낌
- 배열은 참조 변수에 속하기 때문에 참조할 객체가 없다면 `null`을 가진다.

### **배열 선언**

- 배열을 다루기 위한 참조 변수를 선언하는 것
    
    ```java
    int[] score; // 배열 선언
    ```
    

### 배열 생성

- 실제 저장 공간을 생성하는 것
    
    ```java
    score = new int[5]; // 배열 생성
    ```
    

### 2차 배열 선언 방법

1. 타입[][] 변수;
    
    ```java
    int[][] nums;
    ```
    
2. 타입 변수[][];
    
    ```java
    int nums[][];
    ```
    
3. 타입[] 변수[];
    
    ```java
    int[] nums[];
    ```
    
- 여기서 `[]`의 개수가 차원의 수를 의미한다.

## 타입 추론, var

### **타입 추론이란?**

- 변수의 타입을 명시적으로 적어주지 않고도, 컴파일러가 알아서 리터럴을 통해 변수를 추론하는 것이다.
- Java10부터 가능

```java
String word = "hi";

var word = "hi!"; // 컴파일러가 리터럴을 통해서 타입을 추론
```

- `var 타입`은 초기화 없이 사용할 수 없고, null이 들어갈 수 없다.
- 로컬 변수에만 선언이 가능하며 배열을 선언할 땐 var대신 타입을 명시해야 한다.
